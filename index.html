Особенности реализации алгоритмов на последовательном компьютере. Схема устройства однопроцессорного компьютера. Абстрактная модель последовательного компьютера. Проблемы адаптация последовательных алгоритмов на многопроцессорные системы.
Все ныне существующие компьютеры условно можно разделить на 2 группы: последовательные и параллельные
Последовательный компьютер
  • Процессор - выполнение операций
  • Память - хранение информации
  • Процессор - активный
  • Память - пассивный

Между ними канал связи

Работа однопроцессорного компьютера заключается в последовательном выполнении отдельных команд. Упорядоченный набор команд - программа.

Каким бы сложным ни был однопроцессный компьютер, в его основе лежит принцип последовательного выполнения отдельных действий

Для каждого последовательного компьютера время реализации алгоритма пропорционально, главным образом, числу выполняемых операций. => Есть возможность создавать машинно-независимые ЯП.

Algol, Cobol, Fortran, C - примеры таких ЯП.

Порядок выполнения команд всегда является строго последовательным и при заданных входных данных фиксируется однозначно

На самом деле, на одном компьютере параллельно выполняется много различных действий. В таком случае, задача ЯП - скрыть параллелизм.

Почему:
  • I/O
  • Индексы
  • Locks
Примеры:
  • Статистика, аналитика
  • Real-time data
  • Действия пользователей и т.д.
