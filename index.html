OpenNebula — платформа облачных вычислений, предназначенная для управления гетерогенными инфраструктурами распределенных дата-центров. Платформа OpenNebula управляет виртуальной инфраструктурой дата-центра так, что позволяет создавать приватное, публичное или гибридное облако, в результате получая IaaS. OpenNebula — свободное программное обеспечение, лиценция Apache License v2.

История OpenNebula

Структура OpenNebula

OpenNebula управляет хранилищем, сетью, виртуализацией, осуществляет мониторинг и поддерживает различные технологии безопасности, создавая многосвязные сервисы на распределенной инфраструктуре.

Основные компоненты облачной архитектуры

  • Storage
  • Networking
  • Virtualization

Основные компоненты OpenNebula

  • Фронт-енд, который исполняет сервисы ONE
  • Хосты (созданные с помощью гипервизора), которые предоставляют ресурсы, необходимые виртуальным машинам.
  • Хранилище данных, содержащее базовые образы ВМ.
  • Сети -- используются для связи между удаленными серверами

Основные компоненты OpenNebula (2\2)

  • Interfaces & APIs – интерфейсы для взаимодействия с физическими и виртуальными ресурсами
  • User and Groups -- поддержка аккаунтов, групп, авторизации. Разделение прав по типу UNIX.
  • Hosts and Virtualization -- поддержка Xen, KVM, VMware.
  • Networking -- поддержка VLANs и Open vSwitch.
OpenNebula имеет репозиторий образов, репозиторий шаблонов для ВМ и для виртуальных сетей

Структура OpenNebula

OpenNebula Services

  • Демон управления (oned) и планировщик (mm_sched)
  • Демон, отвечающий за мониторинг и за аккаунтинг (onecctd)
  • Веб-интерфейс (Sunstone)
  • Демоны, обеспечивающие доступ к другим облакам (EC2- query или occi)
